Module A · Quality Assessment

Text Dataset Cleaning Pipeline

Step 1

Missing Value Check

Column-wise null check via df.isnull().sum(). Rows with null Hindi or Haryanvi entries dropped.

Step 2

Deduplication

Exact duplicates removed via drop_duplicates() before splitting. Count logged for traceability.

Step 3

Length Filtering

Hindi sentences: min 3 words, max 100 words. Removes fragments and extreme outliers.

Step 4

Manual Spot-check

Subset of XOR-QA pairs reviewed for Haryanvi authenticity. Google Translate artifacts flagged and corrected.

Module A · Splits & Formatting

Dataset Splits & LLM Formatting

Train / Val / Test Split

Train 80% (~4,475) Val 10% (~560) Test 10% (~560)

Leakage Prevention: Deduplication before split. Fixed seed 42. No temporal or source-based leakage.

LLaMA 3.1 Chat Template Format

  • System: Hindi→Haryanvi translator persona with morpho-syntactic rules
  • User turn: Hindi source sentence
  • Assistant turn: Target Haryanvi sentence
  • Max length: 128 tokens, padded, truncated
  • Output: HuggingFace DatasetDict

Adequacy Assessment

Is the Data Sufficient?

Module A — Text

5,594 Parallel Pairs

  • Adequate starting point for LLM fine-tuning on a low-resource dialect task
  • Synthetic data justified — extreme scarcity of Hindi–Haryanvi parallel corpora
  • Bangru morphology enforced in prompt → reduces dialect drift risk
  • If insufficient: Grierson parsing activated as next step
Module B — Audio

5,515 Clips · ~1.89 GB

  • High-quality, pre-aligned baseline from HuggingFace
  • Total usable hours to be confirmed after 1–10s filtering
  • If < 5–10 hrs: Activate YouTube fallback
  • Or: Transfer-learn from Hindi TTS (IndicTTS / Sarvam-TTS)
